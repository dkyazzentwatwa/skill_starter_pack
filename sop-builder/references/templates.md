# SOP Output Templates

Use these as the default structures unless the user requests a different format.

## Standard SOP Template

# [SOP Title]

## Purpose
[What this process accomplishes and why it exists]

## Scope
[What is included, what is excluded, and when to use this SOP]

## Audience / Intended User
[Who this SOP is for and what they are assumed to already know]

## Owner
[Role or team accountable for the SOP]

## Trigger
[What event, request, or condition starts the process]

## Required Inputs / Tools
- [Input or access]
- [Tool or system]
- [Template or prerequisite]

## Procedure
1. **[Step name]** — [Required action and expected result]
2. **[Step name]** — [Required action and expected result]
3. **Decision point:** If [condition], [path A]. If not, [path B].

Mark optional guidance inline as **Optional best practice:** [tip].

## Outputs
- [Primary output]
- [Secondary output or record]

## Quality Checks
- [Check that confirms correctness or completion]
- [Approval or review requirement]
- [What “done” looks like]

## Common Failure Modes
- [Typical error]
- [Missing input or quality issue]
- [Tooling or handoff problem]

## Exceptions / Edge Cases
- [What changes in an unusual case]
- [Alternative path or added requirement]

## Escalation Path
- Escalate when: [condition]
- Escalate to: [role or team]
- Include: [what context, evidence, or links to send]

## Revision Notes
- Last updated: [date]
- Updated by: [name or role]
- Change summary: [what changed]
- Reason: [why it changed]
